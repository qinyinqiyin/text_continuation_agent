# 文本续写助手 - 技术文档

> FastAPI + LangChain 架构说明

---

## 一、项目概述

**文本续写助手** 是基于 **RAG（检索增强生成）** 的智能写作工具：

- 输入前文，AI 自动续写
- 5 种风格：奇幻、古风、科幻、玄幻、悬疑
- 知识库管理：大纲、角色、世界观等设定
- 续写时自动检索知识库并注入 prompt

## 五、续写流程（RAG）

```
1. POST /api/continuation { style, context, requirements, ... }
2. get_agent(style) → RAGAgent
3. RAGAgent.run():
   - retrieve: kb.search_relevant_documents(前文) + story_tools.generate_prompt_fragment(前文)
   - rerank: 通义 gte-rerank 对检索结果重排序（排除 conflict 片段）
   - create_stuff_documents_chain: 将 context + formatted_prompt 送入 prompt
   - create_retrieval_chain: retrieve → combine → llm
   - strategy.post_process(输出)
4. return { success, result }
```

## 六、模块说明

### 6.1 main.py

- FastAPI 应用入口
- RAGAgent 类：create_retrieval_chain 实现 RAG 续写
- 全局单例：get_kb、get_story_tools、get_tool_registry、get_agent
- 全部 API 路由

### 6.2 knowledge_base.py

- **FAISSKnowledgeBase**：通义 embedding + FAISS 向量检索
- 方法：add_setting、search_relevant_documents、search_relevant_settings、get_all_settings、delete_setting、clear_all_settings
- 分段：_simple_split_text（字符）、_split_text_with_bert（token，有 BERT 时）

### 6.3 rerank.py

- **rerank_documents**：通义 gte-rerank 模型对检索结果重排序
- 在 RAG retrieve 环节调用，提升相关文档排序质量

### 6.4 tools.py

- **StoryTools**：冲突检测、状态管理、设定检索
- generate_prompt_fragment：情节限制检查，生成约束片段
- get_story_state、update_story_state、check_consistency、search_lore、search_character

### 6.5 function_call.py

- **create_function_registry**：注册 FilesystemTool、TextAnalysisTool、StoryToolsAdapter
- 供 /api/tools 调用

### 6.6 langchain_llm.py

- **LangChainTongyi**：封装 LangChain 的 Tongyi，对接 DashScope 通义千问

### 6.7 strategies.py

- 5 种风格策略：format_prompt、post_process
- FantasyStrategy、AncientStyleStrategy、SciFiStrategy、EasternFantasyStyleStrategy、SuspenseStrategy

### 6.8 embedding.py

- **get_embedding_model**：通义 text-embedding-v2，1536 维

### 6.9 base_classes.py

- **BaseModel**、**BaseStrategy**：抽象基类
